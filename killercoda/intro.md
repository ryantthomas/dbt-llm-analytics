# dbt + LLM Analytics Lab

You're going to build a real data pipeline, test it, and give it a natural-language interface for Claude.

**What you'll do:**
- Load National Park Service biodiversity data into DuckDB
- Write dbt models that clean and aggregate the data
- Run 18 automated data quality tests
- Wire up an MCP server so Claude can query your database in plain English

**The stack:** dbt, DuckDB, Python, FastMCP

The environment is setting up in the background — Python, dbt, and the lab repo are being installed. This takes about 60 seconds. Start reading and it'll be ready by the time you hit step 1.

Click **START** when you're ready.
